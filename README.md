# 🔍 Image Description Search Bot

Многофункциональный Telegram-бот на базе **Python 3.13+**, предназначенный для умного семантического поиска изображений по их текстовому описанию с использованием AI (Gemini, LlamaIndex, ChromaDB).

## 🚀 Что умеет этот бот

### 1. 🖼 Автоматическое описание изображений
Когда вы отправляете боту фотографию, он с помощью **AI (Gemini)** автоматически генерирует для неё текстовое описание на русском языке. Картинка сохраняется локально, а её текстовое описание и название — в базу данных PostgreSQL.

### 2. 🧠 Умный семантический поиск
Бот позволяет искать картинки по смысловому описанию. С помощью **LlamaIndex** и векторной базы данных **ChromaDB** бот анализирует текстовый запрос (через команду `/search`), находит наиболее подходящие по смыслу описания и отправляет вам соответствующие изображения из локального хранилища.

### 3. 🔄 Синхронизация и индексация
Для ускорения работы поиска используется локальная база ChromaDB. Команда `/sync` позволяет администраторам/пользователям синхронизировать новые фотографии из PostgreSQL в векторную базу. Имеется встроенная защита от дубликатов при индексации.

### 4. 🔐 Система авторизации
Доступ к функционалу бота закрыт. Только авторизованные пользователи, выполнившие команду `/start` и введшие корректный пароль доступа, могут использовать поиск картинок, смотреть статистику и загружать новые фотографии.

## 🛠 Команды бота

### 👤 Основные команды
| Команда | Описание |
| :--- | :--- |
| `/start` | Начать работу с ботом и пройти авторизацию (запрашивает пароль) |
| `/search <запрос>` | Выполнить поиск 5 наиболее подходящих фотографий по текстовому описанию |
| `/sync` | Синхронизировать новые описания изображений из PostgreSQL в ChromaDB |
| `/stats` | Показать подробную статистику по количеству записей в базах PostgreSQL и ChromaDB |

### 📸 Обработка фотографий
| Действие | Описание |
| :--- | :--- |
| **Отправит фото** | Бот примет изображение, сгенерирует к нему AI-описание и добавит в свою базу знаний |

## 🏗 Технологический стек

*   **Ядро**: `Python 3.13+` & `aiogram` (Telegram bot pipeline)
*   **AI (Распознавание)**: `Gemini` через Polza AI API — генерация текстового описания фотографий.
*   **AI (Поиск)**: `LlamaIndex` & OpenAI Embeddings (через API-прокси Aitunnel) — семантический поиск и построение векторов.
*   **База данных (Реляционная)**: `PostgreSQL` (SQLAlchemy 2.0 + asyncpg) — надежное хранение пользователей и истории обработанных картинок.
*   **База данных (Векторная)**: `ChromaDB` — локальное быстрое хранение эмбеддингов для эффективного поиска по смыслу.
*   **Инфраструктура**: `Docker`

## 📥 Полное руководство по установке

### Шаг 1: Скачивание и подготовка

1.  Убедитесь, что у вас установлены [Docker](https://www.docker.com/) и [Git](https://git-scm.com/).
2.  Склонируйте репозиторий:
    ```bash
    git clone https://github.com/ATAGAEV95/Image_description_search.git
    cd Image_description_search
    ```
3.  Создайте файл конфигурации:
    ```bash
    touch .env
    ```

### Шаг 2: Настройка конфигурации (.env)

Откройте созданный файл `.env` и добавьте следующие переменные:

| Переменная | Описание |
| :--- | :--- |
| `TG_TOKEN` | Токен бота от [@BotFather](https://t.me/BotFather) в Telegram |
| `DATABASE_URL` | URL подключения к PostgreSQL (формат: `postgresql+asyncpg://user:pass@host:port/db`) |
| `AI_TOKEN_POLZA` | API-ключ от Polza AI для работы Gemini (распознавание картинок) |
| `AI_TOKEN` | API-ключ (опционально, ProxyAPI) |
| `AI_TOKEN1` | API-ключ от Aitunnel для работы OpenAI Embeddings (LlamaIndex) |

### Шаг 3: Сборка и запуск

#### 🐳 Через Docker (рекомендуется)

1.  **Сборка Docker-образа**:
    ```bash
    docker build -t image-description-search .
    ```
2.  **Запуск контейнера**:
    ```bash
    docker run -d --restart=always \
      --name img-search-bot \
      --env-file .env \
      -v ./chroma_db:/python-app/chroma_db \
      -v ./app/pictures:/python-app/app/pictures \
      image-description-search
    ```
    
> 💡 *Примечание: Примонтированные директории (через флаг `-v`) гарантируют, что загруженные картинки и векторная база ChromaDB не удалятся после перезапуска контейнера.*
